# HPE CPP-3 Project 3  
## Kafka → Logstash Pipeline Setup (Arch Linux)

This document describes the setup and testing of the **Kafka → Logstash ingestion pipeline** used for log processing in the HPE CPP-3 observability project.

The pipeline currently processes logs generated by a simulator, sends them to Kafka, consumes them via Logstash, normalizes the log structure, and outputs the events for debugging.

Future pipeline stages will integrate **VictoriaLogs and Grafana**.

# 7. Kafka Input Configuration

Logstash consumes logs from Kafka topic **logs**.

```bash
input {
    kafka {
        bootstrap_servers => "localhost:9092"
        topics => ["logs"]
        group_id => "logstash-consumer"
        auto_offset_reset => "latest"
    }
}
```

Explanation:

| Parameter | Description |
|--------|-------------|
| bootstrap_servers | Kafka broker address |
| topics | Kafka topic to consume |
| group_id | Consumer group ID |
| auto_offset_reset | Reads latest messages if no offset exists |

# 8. JSON Parsing

Kafka messages are JSON strings.

Example log from generator:

```bash
{
"@timestamp": 1773332384.01865,
"timestamp": "2026-03-12 21:49:44",
"level": "ERROR",
"service": "storage",
"message": "Simulated HPC log event"
}
```


To safely parse the JSON without conflicting with Logstash's internal `@timestamp`, logs are first parsed into a nested object.

```bash
filter {
    json {
        source => "message"
        target => "parsed"
    }
}
```

---

# 9. Field Normalization

Fields are extracted from the parsed object.

```bash
mutate {
    rename => {
        "[parsed][level]" => "level"
        "[parsed][service]" => "service"
        "[parsed][message]" => "message"
        "[parsed][timestamp]" => "timestamp"
        "[parsed][@timestamp]" => "payload_timestamp"
    }
}
```

This prevents Logstash from automatically attempting to parse the incoming `@timestamp`.

---

# 10. Timestamp Normalization

Logs contain two timestamps:

| Field | Format |
|------|--------|
| timestamp | Human-readable |
| @timestamp | Unix epoch |

The pipeline normalizes timestamps using the **date filter**.

```bash
date {
    match => ["timestamp", "yyyy-MM-dd HH:mm:ss"]
    target => "@timestamp"
}
```

Fallback for epoch timestamps:

```bash
if [payload_timestamp] {
    date {
        match => ["payload_timestamp", "UNIX", "UNIX_MS"]
        target => "@timestamp"
    }
}
```

---

# 11. Cleanup

Temporary fields are removed.

```bash
mutate {
    remove_field => ["parsed", "payload_timestamp"]
}
```

# 13. Kafka Consumer Logs


Meaning:

| Log Message | Meaning |
|-------------|--------|
| Adding newly assigned partitions | Kafka assigned partition to consumer |
| Setting offset | Logstash resumed from last committed offset |

# 14. Debugging Techniques

Useful debugging practices:

### Enable structured output

`stdout { codec => rubydebug }`

### Add Kafka metadata

`decorate_events => true`

Adds:

```bash
kafka.topic
kafka.partition
kafka.offset
```

### Test config before running

```bash
logstash --config.test_and_exit -f logstash.conf
```
